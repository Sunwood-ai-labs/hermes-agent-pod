# v0.1.0 リリースノート

![Hermes Agent Pod v0.1.0 release header](/release-header-v0.1.0.svg)

`v0.1.0` は Hermes Agent Pod の初回公開リリースです。`nousresearch/hermes-agent:latest` を sandbox concept から再現しやすいローカル runtime kit にまとめ、kind と Docker Compose の両方、localhost dashboard、Codex 向け worker wrapper を同梱しました。

使い方の流れを追う場合は [v0.1.0 ウォークスルー](/ja/guide/articles/v0-1-0-local-hermes-runtime) を参照してください。

## Highlights

- `sandbox-hermes` kind Pod と `sandbox-hermes-agent` Docker Compose service の 2 経路で Hermes を起動できるようにしました。
- Hermes gateway を `127.0.0.1:8642`、dashboard を `127.0.0.1:9119` に公開し、gateway は local のまま dashboard だけ LAN 内で見る運用を docs に残しました。
- OpenAI-compatible `/v1/chat/completions` 経由で Codex から小さな task を委譲する `scripts/hermes-worker` を追加しました。
- runtime の既定値を Z.AI GLM Coding Plan にそろえ、provider `zai`、model `glm-5.1`、base URL `https://api.z.ai/api/coding/paas/v4` を使う形にしました。
- 英語・日本語の VitePress docs と GitHub Pages deploy workflow を公開面として整備しました。

## Runtime Kit

この release には `compose.yaml`、`kind-config.yaml`、`k8s/` 配下の Kubernetes resources が入っています。Kubernetes 経路では `prompts/hermes-worker-system.md` を `/opt/data/SOUL.md` に反映し、ConfigMap から runtime config を書き込み、agent state を `hermes-data` PVC に保存します。Compose 経路では host 側の `data/` を runtime state として mount します。

どちらの経路も local-first です。gateway と dashboard は既定で localhost に bind し、外部公開には reverse proxy、認証、network control が必要だと docs で明示しています。

## Worker Delegation

`scripts/hermes-worker.py` は Codex 側から使う wrapper です。prompt を引数または stdin から受け取り、`--file` で text file を添付でき、`--session-id` で session continuation に対応します。base URL、API key、model、timeout、system prompt も上書きできます。

設計上の役割分担は狭く保っています。Hermes は API 経由で限定 context を受け取り、host 側の file change と最終判断は Codex が持ちます。

## GLM Defaults And Secret Handling

`scripts/set-glm-key.sh` は Compose runtime 用の GLM key を `data/.env` に保存し、`kind-sandbox-hermes` cluster が見える場合は kind Secret も patch します。実 key は Git に入れず、Secret example は placeholder だけにしています。

runtime defaults は `compose.yaml`、`k8s/hermes-pod.yaml`、`scripts/up.sh` に実装しているため、Compose と kind の両経路で provider、model、base URL、長めの timeout 方針がそろっています。

## Docs And Release Surface

docs site には setup、usage、architecture、troubleshooting の英日 guide が含まれます。dashboard screenshot asset と、VitePress site を GitHub Pages へ publish する GitHub Actions workflow もこの release に入っています。

## Validation

この release note 整備では、tag history を `collect-release-context.ps1` で確認し、SVG asset を `verify-svg-assets.ps1` で検証し、VitePress docs を `npm run docs:build` で build しました。

Docker、kind、live API key が必要な runtime smoke test は、gateway を本格的な委譲に使う前に対象 operator 環境で実行してください。
